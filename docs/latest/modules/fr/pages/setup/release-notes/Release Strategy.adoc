= Stratégie de libération

== Vue d'ensemble

Cette section présente la stratégie de publication de SUSE Observability, y compris ses principaux composants :

* SUSE Observability Platform (Tableau de bord)
* Agents d'observabilité SUSE (tableau de bord)
* Extension de l'interface utilisateur de SUSE Observability Rancher

L'objectif de cette stratégie est de s'assurer que les utilisateurs ont toujours accès aux dernières fonctionnalités, aux mises à jour de sécurité et à la compatibilité entre tous les composants.

== Cadence de libération

Chaque composant suit un modèle de déclenchement par roulement, sans cadence fixe :

* SUSE Observability Platform : Régulièrement mis à jour pour introduire de nouvelles fonctionnalités, des améliorations et des améliorations de la stabilité.
* Agents SUSE Observability : Les versions sont publiées aussi souvent que nécessaire, en particulier pour les correctifs de sécurité et les optimisations de performances.
* SUSE Observability Rancher UI Extension : Mise à jour en même temps que les versions majeures de la plate-forme (Rancher et SUSE Observability) pour maintenir la compatibilité et l'expérience utilisateur.

== Lignes directrices en matière de compatibilité

* Il est toujours recommandé d'utiliser la dernière version de SUSE Observability Agent, car elle inclut des correctifs de sécurité critiques et des améliorations de performances.
* La dernière version de l'agent est compatible avec toutes les versions prises en charge de SUSE Observability Platform.
* Les utilisateurs doivent s'assurer qu'ils exécutent la version recommandée de SUSE Observability Rancher UI Extension correspondant à la version de leur plate-forme.

== Recommandations de mise à niveau

=== Agents SUSE Observability

* Passez toujours à la dernière version pour bénéficier des correctifs de sécurité et des améliorations de performance.
* Les agents sont conçus pour être rétrocompatibles avec toutes les versions de la plate-forme prises en charge.
* Consultez les notes de mise à jour pour avoir une vue d'ensemble des modifications apportées avant la mise à niveau.

=== SUSE Observability Platform

* Passez aux nouvelles versions dès qu'elles sont disponibles pour accéder aux nouvelles fonctionnalités et aux améliorations de la stabilité.
* Examinez les notes de mise à jour pour repérer d'éventuelles ruptures avant de procéder à la mise à niveau.

=== Extension de l'interface utilisateur de SUSE Observability Rancher

* Assurez-vous que la version de l'extension UI correspond à la version recommandée pour votre déploiement Rancher.
** Voir l'extension de l'interface utilisateur et la matrice de compatibilité avec Rancher xref:/k8s-suse-rancher-prime.adoc#_suse_observability_rancher_ui_extension_compatibility_matrix[ici.]
* Les mises à niveau et la compatibilité sont généralement alignées sur les versions des plates-formes Rancher et SUSE Observability.

== Politique de soutien

* Les dernières versions de SUSE Observability Agent sont entièrement prises en charge.
* Les versions précédentes de SUSE Observability Platform bénéficient d'un support pendant une période définie, couvrant généralement la dernière version et les deux versions de correctifs les plus récentes. La mise à jour vers la dernière version est fortement encouragée.
* Les correctifs de sécurité seront appliqués en priorité aux versions les plus récentes de tous les composants.

Pour toute question ou conseil de mise à niveau, veuillez contacter notre équipe d'assistance.
