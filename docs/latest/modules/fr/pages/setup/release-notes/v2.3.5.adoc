== Notes de mise à jour : SUSE Observability Helm Chart v2.3.5

=== Nouvelles fonctionnalités et améliorations

* *Plage de temps de l'interface utilisateur par défaut (configurable) :* L'intervalle de temps par défaut affiché dans l'interface utilisateur a été mis à jour à 1 heure. Cette nouvelle valeur par défaut est conçue pour mieux s'aligner sur les flux de travail courants de dépannage et de surveillance. Les utilisateurs peuvent désormais configurer cette plage de temps par défaut via la valeur `stackstate.ui.defaultTimeRange` Helm. Pour plus de détails, consultez la link:https://github.com/StackVista/helm-charts/tree/master/stable/suse-observability[documentation] SUSE.
* *Configuration de l'affinité Kubernetes :* La génération `suse-observability-values` comprend désormais un fichier supplémentaire, `affinity_values.yaml`, qui gère les paramètres d'affinité de Kubernetes. Pour les profils de haute disponibilité (HA), `requiredDuringSchedulingIgnoredDuringExecution` est désormais utilisé par défaut. Cela garantit que les répliques des services de base de données sont programmées sur différents nœuds, ce qui améliore la résilience. La documentation et les commandes d'installation ont été mises à jour pour refléter ces changements. Pour plus d'informations, consultez la xref:/setup/install-stackstate/kubernetes_openshift/affinity.adoc#_step_3_use_generated_values_in_helm_installation[documentation de SUSE Observability sur Kubernetes Affinity].

== Corrections de bugs

* *StackPack Upgrade Race Condition :* Résolution d'une condition de course qui pouvait se produire lors de la mise à niveau de StackPacks avec plusieurs instances ou configurations. Cette correction permet d'éviter que les instances de StackPack ne restent perpétuellement bloquées dans un état de mise à niveau.
* *Rapport sur les journaux en double :* Correction d'un problème qui entraînait la duplication des journaux.

=== Correction des bugs de l'agent

* *Latence du trafic HTTP WebSocket :* Correction d'un problème où les mesures du trafic HTTP de WebSocket étaient rapportées avec des valeurs de latence anormalement élevées.
