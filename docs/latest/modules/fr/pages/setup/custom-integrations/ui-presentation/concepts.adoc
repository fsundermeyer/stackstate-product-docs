== Présentation des composants

Le concept central du modèle de présentation de l'interface utilisateur est le *ComponentPresentation*.

Un site `ComponentPresentation` définit la manière dont un ou plusieurs composants correspondants sont présentés dans l'interface utilisateur, en fonction du contexte actif.
Il se concentre exclusivement sur les *questions de présentation*, telles que la dénomination, les icônes, les tableaux, les pages de présentation et de mise en évidence, les filtres et l'emplacement des éléments de menu.

À un niveau élevé, une ComponentPresentation :

* Sélectionne les composants à l'aide d'une liaison (une simple requête STQL).
* S'applique dans un mode spécifique (par exemple, l'observabilité).
* Définit un ou plusieurs aspects de la présentation (tels que les tableaux de synthèse ou les points forts).
* Peut définir partiellement les propriétés de présentation et les composer avec d'autres définitions correspondantes.

Plusieurs ComponentPresentations peuvent s'appliquer au même composant. Dans ce cas, leurs définitions sont combinées pour déterminer la présentation finale.
