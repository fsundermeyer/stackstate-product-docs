*Statut :* En cours de développement.

*Disponibilité :* Derrière le drapeau des fonctionnalités de StackPacks 2.0. Pour savoir comment activer cette fonctionnalité, reportez-vous à la section xref:/setup/custom-integrations/overview.adoc[Vue d'ensemble des intégrations personnalisées].

Cette documentation décrit l'intention de conception actuelle et la surface de configuration de haut niveau.

Remarque : aucun comportement ni aucune capacité n'ont encore été mis en œuvre.

Cette fonctionnalité étant en cours de développement, le comportement, les schémas et les capacités peuvent changer au fur et à mesure de l'évolution du modèle.
====

Cette section présente le modèle de *présentation de l'interface utilisateur* introduit dans le cadre des *StackPacks 2.0.*

== Vue d'ensemble

Le modèle de présentation des composants documenté ici déclare comment les composants (par exemple, les pods, les VM, les services) sont présentés dans l'interface utilisateur de {stackstate-product-name}.
Il se concentre exclusivement sur les *questions de présentation*, telles que la dénomination, les icônes, les tableaux, les pages de présentation et de mise en évidence, les filtres et l'emplacement des éléments de menu.

L'objectif de ce modèle est de fournir un moyen plus composable et plus extensible de définir le comportement de l'interface utilisateur pour les composants, par rapport à l'approche existante, plus fragmentée.
