= Guide de démarrage rapide de SUSE Cloud Observability
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Cloud Observability

== Vue d'ensemble

Après avoir acheté SUSE Cloud Observability sur le Cloud Provider Marketplace, votre environnement SUSE Cloud Observability est provisionné.

Vous recevrez un e-mail de SUSE Cloud Observability avec les détails de connexion requis et les liens vers votre environnement. Ce guide de démarrage rapide vous aidera à démarrer et à intégrer vos propres données dans votre déploiement SUSE Cloud Observability.

== Démarrage - Premières étapes

. Configuration d'un mot de passe
. Connectez-vous à votre instance SUSE Cloud Observability
. Installez l'agent SUSE Observability sur votre grappe
. Modifier les données de votre compte personnel

=== Configuration d'un mot de passe

L'e-mail de SUSE Cloud Observability contient un lien unique qui vous permet de définir votre mot de passe initial sur le compte.  Cette opération doit être effectuée avant de pouvoir se connecter et configurer l'environnement d'observabilité.

=== Accès à votre environnement SUSE Observability

Connectez-vous à votre environnement SUSE Cloud Observability en cliquant sur le lien unique figurant dans l'e-mail que vous avez reçu de SUSE Cloud Observability.  En entrant votre mot de passe, vous accéderez à l'écran de configuration afin d'ajouter des grappes.

ifdef::feature-agent-install-cluster[]
=== Installez l'agent SUSE Observability sur votre grappe

SUSE Cloud Observability utilise des StackPacks pour faciliter la configuration de vos grappes en aval et l'intégration des données dans l'environnement d'observabilité.

Pour y accéder, dans l'interface utilisateur de SUSE Observability, ouvrez le menu principal en cliquant en haut à gauche de l'écran et allez à `StackPacks` > `Kubernetes`.

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

Une fois que vous êtes dans l'écran Kubernetes StackPack, il est très simple d'ajouter un cluster à l'environnement d'observabilité.

Entrez un nom pour le cluster que vous souhaitez observer, il ne doit pas nécessairement correspondre au nom du cluster utilisé dans 'kubeconfig'.

Le nom du cluster Kubernetes doit commencer et se terminer par une lettre ou un chiffre minuscule et ne peut être composé que de lettres minuscules, de chiffres, de points et de tirets (. -).

Saisissez le nom et cliquez sur le bouton `Install`.  L'écran suivant devrait s'afficher, le cluster étant marqué comme étant "en attente de données".

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - Ajout d'un cluster]

Cliquez sur le nom du cluster, ce qui développera cette section et révélera une série d'informations comprenant des prérequis et des commandes qui peuvent être utilisées sur vos clusters existants pour les ajouter à votre environnement d'observabilité.

Examinez les informations et assurez-vous que vous disposez des autorisations correctes pour votre environnement Kubernetes et que vous exécutez une version prise en charge de Kubernetes.

Ces commandes sont propres à votre déploiement d'observabilité et comprennent les clés API et les URL nécessaires.  Sélectionnez les commandes appropriées pour votre cluster, il y a des sections pour EKS, RKE, cluster générique Kubnernetes et bien d'autres.

Ces commandes installent les agents SUSE Cloud Obervability et connectent le cluster à votre environnement SUSE Cloud Observability.

Vous pouvez utiliser les commandes directement à partir de l'interface utilisateur ou vous pouvez déployer vers vos clusters en aval en suivant les commandes fournies dans la section "Déployer l'agent StackState et l'agent de cluster" dans le xref:/k8s-quick-start-guide.adoc[guide de démarrage rapide.]

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - Post-installation]

Une fois le cluster connecté, une coche verte doit apparaître dans l'interface utilisateur de SUSE Cloud Observability.

À ce stade, vous pouvez commencer à explorer vos données.
endif::feature-agent-install-cluster[]

=== Modifier les données de votre compte personnel

L'étape 4 de votre courriel consiste à mettre à jour vos informations personnelles.  Cliquez sur le lien unique figurant dans l'e-mail et ajoutez vos données personnelles de base si nécessaire.  Vous pouvez également configurer l'authentification 2FA à partir de cette section si nécessaire.

=== Explorez vos données

Pour commencer à explorer vos données, ouvrez le menu principal en cliquant en haut à gauche de l'écran et allez sur `Kubernetes` pour afficher une liste d'éléments observables.

image::accessing_views_1.png[SUSE Cloud Observability - Exploration de vos données]

Sélectionnez "Clusters" dans la section infrastructure qui devrait afficher une liste des clusters surveillés, sélectionnez votre cluster pour afficher l'une des nombreuses vues intégrées.

image::accessing_views_2.png[SUSE Cloud Observability - Vue des grappes]

À ce stade, vous pouvez commencer à explorer les données de votre cluster ou ajouter d'autres clusters à partir desquels vous pouvez collecter des données.

Pour plus d'informations sur l'utilisation de SUSE Cloud Observability, y compris la création de vues personnalisées, veuillez consulter la documentation standard.

=== Limites de SUSE Cloud Observability

Notez que SUSE Cloud Observability ne fournit pas les signaux RED (Rate, Errors and Duration), seul le signal Rate est affiché.  Cette fonctionnalité est disponible avec SUSE Rancher Prime.
Les clients qui ont besoin de ces signaux pour disposer d'une solution complète d'observabilité doivent contacter SUSE pour discuter de SUSE Rancher Prime. Ces données peuvent également être collectées à l'aide des collecteurs OpenTelemetry.

== Informations complémentaires

Pour plus d'informations, y compris les conditions préalables et les versions de Kubernetes prises en charge pour votre plateforme, veuillez consulter le xref:/k8s-quick-start-guide.adoc[guide de démarrage rapide.]
