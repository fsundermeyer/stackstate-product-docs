== Configurer les notifications Slack

Pour envoyer des notifications à Slack, procédez comme suit :

. <<_connect_slack_workspace,Connectez SUSE Observability à votre espace de travail Slack>>
. <<_select_a_slack_channel,Sélectionnez un canal Slack auquel envoyer les notifications>>
. <<_add_and_test_the_channel,Ajouter et tester le canal>>

=== Connecter l'espace de travail Slack

Cliquez sur le bouton "Choose workspace". Cela ouvrira une page Web Slack où SUSE Observability demande l'autorisation de lister les canaux et d'envoyer des messages aux canaux Slack. Assurez-vous que l'espace de travail Slack souhaité est sélectionné dans le coin supérieur droit.

Cliquez sur "Autoriser" pour continuer.

=== Sélectionner un canal Slack

image::k8s/notifications-slack-channel-configuration.png[Sélectionnez le canal Slack]

De retour dans SUSE Observability, vous pouvez maintenant sélectionner un canal Slack dans la liste. Sélectionnez le canal dans lequel les messages de notification doivent être envoyés.

[NOTE]
====
Les canaux privés ne seront pas listés automatiquement. Pour sélectionner un canal privé, invitez d'abord le robot SUSE Observability dans le canal privé en envoyant un message Slack dans le canal qui mentionne le robot `@SUSE Observability` et en sélectionnant " Invitez-les ". La chaîne sera alors disponible dans la liste.
====


== Ajouter et tester le canal

Enfin, cliquez sur le bouton "Ajouter un canal". Le canal est ajouté à la liste des canaux sur la droite. Un bouton "Test" apparaît. Appuyez sur cette touche pour vérifier que le message de test s'affiche dans le canal Slack sélectionné.

== Messages Slack pour les notifications

Lorsqu'une notification est ouverte, un nouveau message Slack est créé. Ce message sera mis à jour en cas de changement, généralement uniquement lorsque l'état de santé change. Dans le fil de discussion du message, chaque modification apparaîtra également comme un message. Enfin, lorsque la notification est fermée, le message Slack est à nouveau mis à jour (la barre qui indique l'état de santé est désormais grise) et un dernier message est ajouté au fil Slack, qui décrit la raison pour laquelle la notification a été fermée.
