== Vue d'ensemble

En général, le concept de xref:/use/views/k8s-view-structure.adoc[vue] dans SUSE Observability vous permet de surveiller une zone de votre paysage informatique que vous aviez précédemment sauvegardée. Mais souvent, lorsque vous avez besoin d'étudier un sous-ensemble d'une vue particulière, vous ne voulez pas perdre la portée de cette vue. C'est là que les *vues d'exploration* entrent en jeu.

Afin de préserver la portée d'une vue particulière (par exemple `my view`), toutes les actions d'investigation appliquées à un élément topologique ou à une sélection d'éléments (par exemple des composants, des relations, des groupes) s'ouvriront automatiquement dans une vue d'exploration temporaire, sous la vue à partir de laquelle vous avez commencé (par exemple `my view / explore`).

Exemples d'*actions d'investigation* qui seront automatiquement ouvertes dans une vue d'exploration :

* Le * bouton +* a été cliqué sur un composant dans la xref:/use/views/k8s-topology-perspective.adoc[perspective topologique] pour afficher les voisins cachés de ce composant.
* Une *action rapide* a été exécutée sur un composant
* Un élément topologique (composant, relation, groupe) a été *double-cliqué* dans la xref:/use/views/k8s-topology-perspective.adoc[perspective topologique] pour l'étudier.
* Le lien ou le bouton *"Explorer..."* a été cliqué sur un élément topologique (composant, relation, groupe) pour l'étudier.

[TIP]
====
Les actions d'investigation exécutées à partir d'une vue d'exploration existante *n'* ouvriront *pas* une nouvelle vue d'exploration - elles modifieront la portée de la vue d'exploration dans laquelle vous vous trouvez.
====


== Manipulation

Lorsqu'une vue d'exploration est automatiquement créée, son champ d'application initial est également défini. Cette fonction est utile si vous souhaitez rétablir la portée initiale de la vue d'exploration après y avoir apporté des modifications.

Bien qu'une vue d'exploration soit censée être temporaire, elle peut être enregistrée en tant que xref:/use/views/k8s-custom-views.adoc[vue personnalisée] si vous souhaitez la contrôler ou la revoir ultérieurement.

Si vous ne souhaitez pas sauvegarder une vue d'exploration, il vous suffit de vous en éloigner ou de revenir à la vue de départ en utilisant les fils d'Ariane de la barre de navigation supérieure.

== Structure

Les vues explorées ont une structure identique à celle des xref:/use/views/k8s-custom-views.adoc[vues personnalisées]: elles ont les mêmes xref:/use/views/k8s-view-structure.adoc#_filters[filtres] et les mêmes xref:/use/views/k8s-view-structure.adoc#_perspectives[perspectives].
