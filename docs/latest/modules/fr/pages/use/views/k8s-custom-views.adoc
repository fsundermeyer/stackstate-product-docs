== Vue d'ensemble

Alors qu'une xref:/use/views/k8s-views.adoc[vue kubernetes] sert de point de départ à l'exploration d'une partie spécifique de votre paysage informatique, et qu'une xref:/use/views/k8s-explore-views.adoc[vue d'exploration] vous permet d'étudier un sous-ensemble d'une vue particulière, une *vue personnalisée* vous permet de revenir à l'une ou l'autre de ces vues.

En d'autres termes, vous pouvez créer une vue personnalisée en enregistrant une xref:/use/views/k8s-views.adoc[vue kubernetes] ou une xref:/use/views/k8s-explore-views.adoc[vue explore] avec vos propres paramètres (filtres, options de visualisation, paramètres d'affichage, configuration de la chronologie) pour mettre en signet une partie de votre topologie qui présente un intérêt particulier pour vous ou votre équipe.

[NOTE]
====
Par défaut, les vues personnalisées sont visibles par tous les utilisateurs. Elles peuvent être sécurisées par un *administrateur de SUSE Observability* si nécessaire.
====

== Manipulation

Toutes les vues personnalisées se trouvent sous l'entrée *Vues* dans le menu principal.

Si vous utilisez fréquemment une certaine vue personnalisée, vous pouvez l'ajouter aux *vues marquées d'un astérisque*. Toutes les vues étoilées sont listées directement dans la section *Vues* du menu principal pour un accès rapide - vous pouvez reconnaître les vues étoilées par l'icône de l'étoile jaune à côté de leur nom. Ajoutez ou supprimez une vue personnalisée des vues étoilées en cliquant simplement sur l'icône en forme d'étoile située à côté de son nom.

=== Créer une vue

Pour créer une nouvelle vue personnalisée, cliquez sur le bouton bleu *Save view as...* dans la barre de navigation supérieure lorsque vous êtes sur une xref:/use/views/k8s-views.adoc[vue kubernetes] ou une xref:/use/views/k8s-explore-views.adoc[vue explorer]. Pour créer une nouvelle vue à partir d'une vue personnalisée modifiée, utilisez le menu déroulant à côté du bouton et sélectionnez *Enregistrer la vue comme...*.

image::k8s/k8s-custom-view-edit-settings.png[Modifier les paramètres d'affichage]

Dans la boîte de dialogue *Enregistrer la vue sous*, les options suivantes peuvent être définies :

|===
| Nom du champ | Description

| Voir le nom
| Le nom de la vue.

| Description
| Une brève description de la vue. Cela permet de se souvenir de la portée de la vue.

| Identifiant
| Une référence unique pour la vue, utile lorsque vous souhaitez référencer la vue à partir d'une configuration exportée, telle que la configuration exportée dans un StackPack.
|===

=== Réinitialiser une vue

Lorsqu'une vue personnalisée est créée, tous les filtres, les options de visualisation, les paramètres de la vue et la configuration de la ligne de temps sont enregistrés dans la vue. Si vous souhaitez rétablir l'état initial de la vue personnalisée après y avoir apporté des modifications, utilisez le menu déroulant situé à côté du bouton *Enregistrer...* dans la barre de navigation supérieure et sélectionnez *Réinitialiser la vue.*

=== Supprimer ou modifier une vue

[NOTE]
====
Notez que les modifications apportées à une vue personnalisée seront appliquées à tous les utilisateurs.
====


Une vue personnalisée peut être modifiée ou supprimée à partir de l'écran " *Toutes les vues"* ou de l'écran " *Vue personnalisée"*.

. Écran de *toutes les vues*:
 ** Cliquez sur *Vues* dans le menu principal pour ouvrir l'écran de *toutes les vues*.
 ** Survolez la vue que vous souhaitez modifier ou supprimer.
 ** Cliquez sur le bouton *Modifier les paramètres d'affichage* ou *Supprimer l'affichage* à droite.
. Écran de *visualisation personnalisé*:
 ** Ouvrir la vue.
 ** Ouvrez le menu déroulant situé à côté du bouton *Enregistrer...* dans la barre de navigation supérieure.
 ** Cliquez sur *Modifier les paramètres de la vue* ou sur *Supprimer la vue*.

== Structure

Une vue personnalisée a une structure identique à la xref:/use/views/k8s-views.adoc[vue kubernetes] ou à la xref:/use/views/k8s-explore-views.adoc[vue explore] à partir de laquelle elle a été créée : elle a les mêmes xref:/use/views/k8s-view-structure.adoc#_filters[filtres] et les mêmes xref:/use/views/k8s-view-structure.adoc#_perspectives[perspectives].
