= Pour commencer
:revdate: 2025-07-10
:page-revdate: {revdate}

Bonjour ! Vous avez donc intégré vos clusters Kubernetes ou OpenShift et vous êtes prêt à démarrer.

Après avoir configuré votre xref:/k8s-quick-start-guide.adoc[intégration avec Kubernetes], vous pouvez ouvrir le menu principal pour explorer vos ressources. Vous pouvez par exemple commencer par les services.

== Explorez vos ressources Kubernetes

image::k8s/k8s-quick-start-menu.png[Menu principal]

Vous accédez ainsi à la vue d'ensemble des services, qui affiche tous les services en cours d'exécution dans vos clusters. Si vous cliquez sur l'un des autres éléments sous Kubernetes, vous accéderez à la page d'aperçu de ce type de ressource. Il affichera d'abord toutes les ressources de ce type dans tous les clusters et tous les espaces de noms.

image::k8s/k8s-quick-start-services.png[Aperçu des services]

En haut à droite, vous avez la possibilité de filtrer votre sélection sur un certain cluster et/ou espace de noms pour voir les ressources dont vous êtes responsable.

En bas à gauche, vous trouverez deux entrées.

. Le sélecteur de plage de temps. Cela permet de sélectionner la période de temps pour toutes les mesures, tous les journaux et tous les événements que vous voyez dans le produit.
. Le sélecteur de temps topologique. Il permet de revenir à un certain moment dans le temps pour voir l'état exact de vos systèmes tel qu'il a été observé à ce moment-là.

Vous pouvez par exemple filtrer sur un certain espace de noms, dans ce cas, je filtre les services jusqu'à "sock-shop" qui est une application de démonstration utilisant différents microservices écrits dans différents langages de programmation et utilisant différents moyens de communication pour servir d'exemple pour la résolution d'un problème.
Si vous cliquez maintenant sur Topologie, vous verrez la topologie des composants actuellement sélectionnés (dans ce cas, les services de la boutique en ligne).

image::k8s/k8s-quick-start-service-topology.png[Topologie des services]

Dans la topologie, vous voyez toutes les ressources, dans ce cas les services.

* Si vous cliquez sur un composant (dans ce cas, un service), les détails du service s'affichent, y compris les mesures les plus importantes, par exemple, la latence, le débit et le taux d'erreur dans le cas d'un service. À côté des mesures les plus importantes, l'état de santé du composant est affiché et développé si quelque chose ne va pas.
* Si vous cliquez sur une relation, vous verrez le détail de la relation, y compris tous les composants qui en font partie. Dans le cas d'une carte de services, vous verrez tous les composants impliqués dans la communication de service à service.
Si vous souhaitez ouvrir un composant pour voir tous les détails de cette ressource (par exemple, les détails de ce service ou d'un certain service), vous pouvez cliquer sur le bouton "Ouvrir le composant" à partir d'un composant sélectionné (qui apparaît alors dans le panneau latéral droit) ou vous pouvez ouvrir le composant en cliquant sur le nom du composant dans l'onglet de la page d'aperçu.

image::k8s/k8s-quick-start-service.png[Aperçu des services]

Après avoir ouvert une ressource Kubernetes, vous obtiendrez une perspective Highlight qui vous montrera tous les points forts de ce composant.

. Les méta-données du composant
. Les actions disponibles sur le composant, dans le cas d'un service, cela vous donne la possibilité d'afficher l'état et/ou les informations de configuration. Si vous voulez voir les logs, vous pouvez ouvrir les pods via les ressources associées qui vous donnent accès aux logs.
. Ressources connexes. Cette section montre toutes les ressources liées à cette ressource, en l'occurrence 2 autres services avec lesquels elle communique et 1 pod qui soutient ce service.
. La section des moniteurs affiche tous les moniteurs appliqués à cette ressource, y compris leur état à l'instant topologique sélectionné.
. La section des mesures vous indique toutes les mesures importantes pour ce service. Les mesures comprennent l'intervalle de temps de télémétrie sélectionné.
. La ligne de temps de l'état de santé d'un service montre l'état de santé de cette ressource au fil du temps.
. Une chronologie des événements montrant tous les événements survenus sur ce service au fil du temps.

Explorons maintenant un moniteur déclenché en cliquant sur le moniteur "HTTP - 5xx error ratio".

image::k8s/k8s-quick-start-service-5xx-error-triggered-monitor.png[HTTP - 5xx ratio d'erreur déclenché par le moniteur]
